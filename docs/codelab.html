1. Introduction & Set Up

Welcome to the Juneteenth Build Jam! In the next 75 minutes, you are going to design, build, and deploy a real, working AI-powered web application using Antigravity, an agentic coding assistant.

What You'll Build

An AI-powered web application based on one of our five local civic tracks. The app runs in your browser, interfaces with Google's Gemini API, and is fully customizable. You direct the AI, and the AI writes the code.

Install Antigravity

Antigravity is the agentic companion that will do the heavy lifting. Install it on your machine first:

  1. Go to https://antigravity.google/download in your web browser.
  2. Click the download button for your operating system (macOS, Windows, or Linux) to get the installer for Antigravity 2.0.
  3. Run the installer and click through the prompts using the default settings.

Launch & Sign In

  1. Open the Antigravity application from your applications directory.
  2. Click "Sign In" when prompted.
  3. Use your Google account. (Tip: Use the same Google account you plan to use for Google AI Studio later on).

Verify the Welcome Screen

Once signed in, you should see the welcome screen presenting three primary buttons:

  • Open Folder: Opens an existing directory as a workspace. (This is the one we'll use.)
  • Clone Repository: Downloads a codebase directly from GitHub.
  • Start without a folder: Jumps straight into the agent. You won't need this one today.
Checkpoint

Antigravity is successfully installed, you are logged in, and the welcome screen is open.

2. Install Tools & Get the Code

We need two tools installed — uv (runs Python apps) and git (downloads code) — plus the event repo cloned to your machine. Pick the path that fits you:

⭐ Option A — Let Antigravity do everything (recommended for beginners)

Stay inside Antigravity the whole time. No terminal needed.

  1. On the Antigravity welcome screen, click "Open Folder".
  2. Create an empty scratch folder on your Desktop (e.g., jam-setup) and open it. This folder is only there so Antigravity has a chat open — you'll switch to your real track folder in Step 3. The chat interface will load.
  3. Select Gemini 3.5 Flash (Medium) from the model dropdown.

4. The Spec Talk

        
Before the AI writes a single line of code, you and the AI get on the same page about what you're building. This takes about 5 minutes and saves a lot of backtracking later.


        
Open the Chat

        
Click inside the chat area at the bottom of Antigravity (Ctrl+L on Windows/Linux, Cmd+L on Mac). Make sure Gemini 3.5 Flash (Medium) is selected in the model dropdown. Then paste this:


        📝 Paste this:
        

          
          
Read @SPEC.md and ask me 3 short questions before we start building:
1. What's the one thing a user types in or uploads?
2. What should they see when it works — the magic moment?
3. What are we NOT building today to keep it simple?

Don't write any code yet. Just ask the questions and wait for my answers.

        


        
Answer the Questions

        
The AI will ask you three questions. Answer them in your own words. There's no wrong answer — you're the product owner. For example, for the Stand Up Shop track:

        
    
          
  1. "Our neighborhood association wants to turn an abandoned lot into a community garden..."
    2. 
          
  2. "The magic moment is when it finds a real grant we qualify for."
    3. 
          
  3. "No payments, no databases — just text in, AI output out."
    4. 
        


        

          
💡 Tip — Narrow scope = faster ship

          
The tighter your answers, the faster you'll have a working app. You can always add features later. Today's goal is something real that runs.

        


        

          
✅ You're good to move on when...

          
The AI has asked its three questions and you've answered them. No code written yet — that comes next.

        

      


      
      

        
5. Generate 3 Design Docs

        
Now you'll have the AI plan the entire app before it writes code. Think of these three files as your blueprint — they keep the AI focused so it doesn't go off and build the wrong thing.


        
    
          
  • 📋 product.md — what the app does and who it's for
    • 
          
  • 🎨 ui.md — what it looks like and how users interact with it
    • 
          
  • ⚙️ engineering.md — the tech stack, file structure, and test plan
    • 
        


        
Switch Models

        
In the model dropdown, switch to Gemini 3.1 Pro (Low). This gives the AI more reasoning power for planning work.


        📝 Paste this:
        

          
          
Based on our conversation and @SPEC.md, create three planning files in this folder:

1. product.md — what the app does, who it's for, and the key features
2. ui.md — how the interface looks: layout, colors, fonts, and user flow
3. engineering.md — the tech stack, file structure, and a testing plan
   (use Python + FastAPI for the backend, plain HTML/CSS/JS for the frontend)

Save each file before moving to the next. After all three are saved,
give me a quick summary of what's in each one.

        


        
Wait for the AI to finish. It'll confirm each file as it saves it.


        
Review the Docs

        
The AI will create each file as an Artifact — you'll see them appear in the Agent Manager panel as it works. Click each one to read it. If something looks off — wrong colors, wrong features — just tell the AI in plain English and it'll update the doc. Don't touch the code yet.


        

          
💡 Tip — The doc is the source of truth

          
If the app doesn't look right later, come back and fix the doc first. Then ask the AI to rebuild. Changing the plan is faster than patching code.

        


        

          
✅ You're good to move on when...

          
All three files — product.md, ui.md, engineering.md — appear as Artifacts in Agent Manager and look like a solid plan for your track.

        

      


      
      

        
6. Build & Test

        
This is the big one. The AI reads your three docs and builds the whole app — backend, frontend, and tests. Your job is to click Allow when it asks permission to run commands.


        
Switch to the Thinking Model

        
Switch to Gemini 3.1 Pro (High) in the model dropdown. This model reasons through the full architecture before it starts writing — fewer mistakes, better code.


        📝 Paste this:
        

          
          
Build the full app based on @product.md, @ui.md, and @engineering.md.

- Use Python + FastAPI for the backend, plain HTML/CSS/JS for the frontend
- Use uv for package management (not pip directly)
- Read the Gemini API key from a .env file as GEMINI_API_KEY
- Do NOT start any servers yet — I'll do that in the next step

After the code is written:
- Write tests for the main functions using pytest
- Mock the Gemini API in tests so they run without a real key
- Run the tests and fix any failures (try up to 3 times)
- Tell me when everything passes or what's still broken

Click Allow when I get permission prompts for installing packages or running tests.

        


        
Sit back and watch it work. Click Allow on any permission pop-ups that appear — these are just Antigravity asking before it runs terminal commands.


        

          
✅ You're good to move on when...

          
The AI reports something like All tests passed and you can see new files as Artifacts in Agent Manager (an app.py or similar, plus a tests/ folder).

        


        

          
🚨 If something goes wrong

          
If tests keep failing after 3 tries, paste this into the chat:


          "Stop and explain what's broken. What do I need to change in engineering.md to fix this?"


          Update the doc based on its answer, then ask it to try building again.

        

      


      
      

        
7. Add Your Gemini Key

        
The app is built but it's running on a fake AI connection. Now we plug in your real Gemini API key so it talks to the actual model.


        
Get Your Key

        
    
          
  1. Open aistudio.google.com/app/apikey in your browser.
    2. 
          
  2. Sign in with your Google account.
    3. 
          
  3. Click "Create API key" → create it in a new or existing project.
    4. 
          
  4. Copy the key — it starts with AIzaSy...
    5. 
        


        
Add it to the App

        
Paste this into the Antigravity chat, replacing the placeholder with your actual key:


        📝 Paste this (swap in your real key):
        

          
          
Create a .env file in the project root with this content:
GEMINI_API_KEY=<PASTE_YOUR_KEY_HERE>

Also:
- Make sure .env is listed in .gitignore so the key never gets pushed to GitHub
- Confirm the app reads GEMINI_API_KEY at startup using os.getenv()
- Show me the .env file with the key hidden (first 6 characters + ...)

        


        

          
🔑 Keep your key private

          
Never paste your API key directly into the chat or commit it to GitHub. It always lives in .env, which stays on your machine only.

        


        

          
✅ You're good to move on when...

          
The AI confirms your .env is set up and shows you something like GEMINI_API_KEY=AIzaSy... with the rest hidden.
